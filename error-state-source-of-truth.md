# Error State Source Of Truth

## Summary

- Scope: Centralized read-only audit of error states, validation messages, loading failures, empty states, and related user-facing copy found in the two connected Bizko Asana boards.
- Asana projects searched:
  - Bizko - Initial Release (`1214215045871819`): https://app.asana.com/1/1210495714634447/project/1214215045871819
  - Bizko - Design Team (`1214163061817187`): https://app.asana.com/1/1210495714634447/project/1214163061817187
- Search keywords used: error, error state, validation, invalid, required, failed, unable, cannot, permission, timeout, retry, not found, empty state, loading.
- Last updated: 2026-07-14
- Coverage notes: This report uses targeted Asana MCP searches, then detail reads only for relevant candidate tasks in the two scoped projects. Findings include exact copy where visible in Asana descriptions/comments. When Asana referenced screenshots, Figma, or external evidence without quoting the copy, the message is marked `Not specified`.
- Known gaps: This pass did not broadly crawl every task in either project and did not open external Drive/Figma evidence. Some rows represent empty/loading states because they were returned by the requested keywords and affect user-visible error-state behavior.

## Rate Limit Note

Asana MCP/API reads were kept below 60 requests per minute. The pass used 14 keyword searches plus targeted task detail reads, with no Asana writes.
